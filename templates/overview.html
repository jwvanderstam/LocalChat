Detailed Architecture & Functions
Overview

LocalChat is a production-ready RAG system built with Flask, Ollama, PostgreSQL (pgvector), and an optional Redis cache. The application uses the factory pattern with clean blueprint separation and full observability.

Core Components
  • Flask App Factory: src/app_factory.py
  • RAG Engine: src/rag/ (chunking, retrieval, scoring, cache, web_search)
  • Database Layer: src/db/ (PostgreSQL + pgvector, HNSW, connection pool)
  • Ollama Client: src/ollama_client.py
  • Monitoring: src/monitoring.py (Prometheus, health, request timing)
  • Security: src/security.py (JWT, rate limiting, CORS)
  • Settings Routes: src/routes/settings_routes.py
  • Tool System: src/tools/ (registry, executor, builtins)
  • Cache: src/cache/ (memory / Redis / database backends)
  • Validation: src/models.py (8 Pydantic models)
  • Exceptions: src/exceptions.py (11 custom types)
Key Metrics
  • 1,094 tests — 81% coverage (SonarCloud verified)
  • Streaming via Server-Sent Events
  • HNSW approximate nearest-neighbour search
  • Hybrid retrieval: semantic cosine + BM25 keyword
  • Multi-signal reranking (score fusion)
  • Connection pooling (2–10 connections)
  • Parallel ingestion (8 workers, 512 chunks/batch)
  • GPU auto-detection: NVIDIA & AMD
  • Prometheus metrics + JSON metrics endpoint
  • X-Request-Duration header on every response
Data Flow — Query
  1. User sends question → Flask receives via SSE route
  2. Question → Ollama → query embedding vector
  3. pgvector HNSW search → top-30 candidate chunks
  4. Hybrid re-rank (BM25 + semantic) → top-12 chunks
  5. Context + conversation history → LLM prompt
  6. Ollama streams tokens → SSE to browser
Data Flow — Ingestion
  1. File uploaded → format loader (PDF/DOCX/TXT/MD)
  2. Text extraction with table preservation
  3. Chunker splits at sentence boundaries (1,200 chars / 150 overlap)
  4. 8 parallel workers → Ollama embeddings (512 chunks/batch)
  5. SHA-256 fingerprint for duplicate detection
  6. Bulk insert into PostgreSQL + pgvector HNSW index
Performance Notes
  • Document processing: ~5 s per MB
  • Query response: <1 s (retrieval) + LLM generation time
  • Full test suite: ~18 s (1,094 tests)
  • GPU queries (nvidia-smi/rocm-smi) cached 30 s; Ollama /api/ps cached 5 s
